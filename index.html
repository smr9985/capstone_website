Scoring Tracking Association using MLPs

Overview

This page contains information for a capstone project, done as part of the requirements for the Computer Science MS program at RIT.

The goal of this project is to develop and evaluate an intelligent radar tracking system for converting noisy detection reports into continuous target tracks. The system uses Kalman filters for motion prediction and a learned MLP association model to combine spatial, uncertainty-based, reflectivity-based, radial-velocity-based, and track-maturity features when matching detections to existing tracks.

General Background

Multi-Object Tracking

Multi-object tracking estimates the state and identity of multiple targets over time. A common tracking pipeline follows a predict, associate, and update cycle: existing tracks are predicted forward, new detections are associated with those tracks, and matched tracks are updated using the assigned detections.

Monostatic Radar

In a monostatic radar system, the transmitter and receiver are located at the same site. After radar processing, a possible target can be represented as a detection report. A detection report may include measurements such as estimated position, radial velocity, and received signal strength or radar cross section.

Tracking Strategies

Tracking systems can generally be separated into two broad association strategies: hard assignment methods and probabilistic assignment methods. Hard assignment methods choose a single best detection for each track, while probabilistic methods consider multiple possible detections and weight them by likelihood.

Global nearest neighbor, or GNN, is a hard assignment approach that selects a one-to-one matching between existing tracks and new detections. Joint probabilistic data association, or JPDA, is a probabilistic approach that updates tracks using association probabilities across multiple feasible detections.

Learned Association Model

The learned model in this project follows a GNN-based tracking structure because the final association step still produces a one-to-one assignment. The MLP is used to score candidate track-detection pairs before the final assignment is selected, allowing the tracker to combine multiple association cues while keeping the structure of a hard-assignment tracker.

Helpful Resources

RadarSimPy

RadarSimPy Documentation

RadarSimPy is an open-source radar simulation library for modeling radar systems and target returns. In this project, it was used to simulate the radar power return used in the generated detection reports.

Stone Soup

Stone Soup Documentation

Stone Soup is an open-source Python framework for tracking and state estimation. In this project, it was used to implement out-of-the-box GNN and JPDA tracking baselines, allowing the learned tracker to be compared against established tracking methods.

PyTorch

PyTorch Documentation

PyTorch is an open-source machine learning framework used for building and training neural networks. In this project, it was used to train the MLP model that scores candidate track-detection associations.

FilterPy

FilterPy Documentation

FilterPy is a Python library for Kalman filtering and Bayesian filtering. In this project, it was used to support the Kalman filter prediction and update steps used inside the custom tracking framework.
